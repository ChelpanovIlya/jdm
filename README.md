# Data Model Designer Prompt

Коллекция промптов, схем и правил для генерации строго типизированных логических моделей данных с помощью LLM.

## Быстрый старт
1. Откройте чат с LLM, например https://chat.qwen.ai/ или https://chat.deepseek.com/
1. Скопируйте содержимое `PROMPT.md` в поле ввода чата.
2. Опишите задачу проектирования. Вы можете описать задачу в общем или детально. Пример общей задачи: "спроектируй модель для интернет магазина". Пример задачи средней степени детализации: "спроектируй модель банковской системы с сущностями "Клиент", "Адрес", "Счет" и "Валюта"
3. Получите в ответ: доменную модель в формате JSON (JDM) и Mermaid-диаграмму.
4. Проверьте модель на корректность и соответствие требованиям.
5. Для корректировки модели давайте команды в чате на русском языке и оценивайте результат в текстовой или графической форме. Например, вы можете дать команду: добавь счету атрибут "остаток в национальной валюте" или добавь клиенту код идентификации юридических лиц LEI. Тип данных, размерность, название ИИ предложит сам. Альтернативный вариант - скопируйте JDM и внесите в него изменения. Результат отправьте в чат. LLM примет изменения и дальнейшая работа будет продолжена с этой версией.
6. Когда вы будете удовлетворены результатом, сохраните JDM в файл для дальнейшего использования.
7. Переходите к генерации физической модели данных, демо данных и проверки модели SQL запросами к демо данным.

## Генерация физической модели данных в виде DDL скрипта и демо данных
1. Скопируйте содержимое `prompts/ddl-prompt.md` в чат. Можно сразу дополнить требованиями к объему демо-данных. Например, сгенерируй демо данные для 5 клиентов со счетами в основных валютах. в позициях 6-8 счета используй цифровой код валюты в соотвествии с правилами учета в РФ.
2. Получите в ответ: ddl скрипт для создания БД для PostgreSQL и скрипт заполнения БД демо-данными.

## Генерация запросов по демо данным
1. Дайте команду "создай SQL запросы для проверки данных" или "создай SQL запрос возвращающий сумму завершенных заказов в разрезе клиентов с начала 2026 года"
2. Получите в ответ SQL запросы

## Внесение изменений в существующую модель 
1. Откройте час с LLM и задайте промпт `PROMPT.md`
2. Отправьте полную JDM или подмножество сущностей модели в чат.
3. Опишите задачу проектирования, например: добавь сущность "Курс валюты"
